§ String rotations

Given a string sss of length nnn, we can index it as s[0]s[0]s[0], s[1]s[1]s[1], upto s[n−1]s[n-1]s[n−1]. We can now build a table consisting of rotations of the string. We'll define: 
    G[q](ηo(id_o)) = ηo(q'(id_o))
    G[q](ηo(id_o)) = ηo(q' . id_o)
    G[q](ηo(id_o)) = ηo(q')
    ηo(q') = G[q](ηo(id_o))
We can build a table of left-rotations: 
    G[q](ηo(id_o)) = ηo(q'(id_o))
    G[q](ηo(id_o)) = ηo(q' . id_o)
    G[q](ηo(id_o)) = ηo(q')
    ηo(q') = G[q](ηo(id_o))
We can immediately notice that it's a symmetric matrix. We can prove this as follows. We write lrot(rot,s)lrot(rot, s)lrot(rot,s) as an array such that:
lrot(rot,s)[i]=s[(rot+i) lrot(rot, s)[i] = s[(rot + i) % n] \text{where $n = |s|$} lrot(rot,s)[i]=s[(rot+i)
Now, we note that on row rrr of our array we have the string lrot(r,s)lrot(r, s)lrot(r,s). So the value at row rrr, column ccc is lrot(r,s)[c]=s[(r+c)lrot(r, s)[c] = s[(r + c)%n]lrot(r,s)[c]=s[(r+c). But this is symmetric in c,rc, rc,r so can be written as s[(c+r)s[(c + r)%n]s[(c+r), which is equal to lrot(c,s)[r]lrot(c, s)[r]lrot(c,s)[r]. Formally:
lrot(r,s)[c]=s[(r+c) lrot(r, s)[c] = s[(r + c) %n] = s[(c + r)%n] = lrot(c, s)[r] lrot(r,s)[c]=s[(r+c)

§ Sorts of string rotations

We're next interested in considering sorted order of the string rotations. For example, in the case of the string "here, there", all the rotations are: 
    G[q](ηo(id_o)) = ηo(q'(id_o))
    G[q](ηo(id_o)) = ηo(q' . id_o)
    G[q](ηo(id_o)) = ηo(q')
    ηo(q') = G[q](ηo(id_o))
which is calculated with the help of the following haskell code: 
    G[q](ηo(id_o)) = ηo(q'(id_o))
    G[q](ηo(id_o)) = ηo(q' . id_o)
    G[q](ηo(id_o)) = ηo(q')
    ηo(q') = G[q](ηo(id_o))
If we now sort these rotations, we get: 
    G[q](ηo(id_o)) = ηo(q'(id_o))
    G[q](ηo(id_o)) = ηo(q' . id_o)
    G[q](ηo(id_o)) = ηo(q')
    ηo(q') = G[q](ηo(id_o))
we produce this by chainging the above definition of main to: 
    G[q](ηo(id_o)) = ηo(q'(id_o))
    G[q](ηo(id_o)) = ηo(q' . id_o)
    G[q](ηo(id_o)) = ηo(q')
    ηo(q') = G[q](ηo(id_o))
Let's look at the final column of that table. We have: 
    G[q](ηo(id_o)) = ηo(q'(id_o))
    G[q](ηo(id_o)) = ηo(q' . id_o)
    G[q](ηo(id_o)) = ηo(q')
    ηo(q') = G[q](ηo(id_o))
Now, we'll show how to write a really cool function call bwt such that: 
    G[q](ηo(id_o)) = ηo(q'(id_o))
    G[q](ηo(id_o)) = ηo(q' . id_o)
    G[q](ηo(id_o)) = ηo(q')
    ηo(q') = G[q](ηo(id_o))
The 5 is the index of the original string in the list. That is, given the jumbled up last-column and the index of the original string, we're able to reconstruct the original string. The reason this is useful is that the jumbled string "errhhetee-" is easier to compress: it has long runs of r, h, and e which makes it easier to compress. The process we performed of rotating, sorting the rotations and taking the final column is the Burrows-Wheeler transform. in code: 
    G[q](ηo(id_o)) = ηo(q'(id_o))
    G[q](ηo(id_o)) = ηo(q' . id_o)
    G[q](ηo(id_o)) = ηo(q')
    ηo(q') = G[q](ηo(id_o))
Now we need to understand how bwtinv is defined and what it does. The definition is: 
    G[q](ηo(id_o)) = ηo(q'(id_o))
    G[q](ηo(id_o)) = ηo(q' . id_o)
    G[q](ηo(id_o)) = ηo(q')
    ηo(q') = G[q](ηo(id_o))
OK, so much for the code. what does it do? The idea is that:
  • we recreate the entire matrix from the last column using recreate and take the kth element.
  • recreate apprents a copy of the initial last column to a matrix of columns, and then sorts this.
